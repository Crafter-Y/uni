## Servlets

HTTP:

- Anfragen vom Client, Antworten vom Server
- Der server agiert nicht von sich aus
- Keine Antwort ohne vorherige Frage

Was macht ein Webserver?

- Auf Verbindungen warten
- Request entgegennehmen
- Request interpretieren
- Anhand der URI dem zuständigen Programmcode zuordnen
- Parameter auslesen
- Antwort erzeugen
- Inhalte zurücksenden
- Entgegennahme von HTTP Anfragen
- Abarbeiten mehrerer Anfragen gleichzeitig
- Vorverarbeitung von Anfragen

Es gibt mehrere historische Ansätze, dieses Konzept umzusetzen:

- SSI (Server Side Includes) - Ein Prozess pro Anfrage, Rückgabe durch Code in HTMl Kommentaren
- CGI (Common Gateway Interface) - Aufruf eines Scripts pro Anfrage

Als nächsten Schritt hat man die Schnittstelle in den Webserver mit eingebaut, z.B. als Modul für Apache: PHP; PERL, PYTHON, WEBDAV

Vorteile:

- Performanz
- Mächtige Programmiersprachen nutzbar
- Plattformunabhängig

Nachteile:

- Eine Anfrage kann den ganzen Server abstürzen lassen
- Mehr Sicherheitsherausforderungen

Wir werden uns mit Java Servlets befassen.

Ein Servlet bezeichnet man Java Klassen, deren Instanzen innerhalb eines Webservers Anfragen von Clients entgegennehmen und beantworten.

Ein HttpServlet hat folgende Methoden, die überschrieben werden können:

- `doGet(HttpServletRequest req, HttpServletResponse resp)`
- `doPost(HttpServletRequest req, HttpServletResponse resp)`
- `doDelete(HttpServletRequest req, HttpServletResponse resp)`
- `doPut(HttpServletRequest req, HttpServletResponse resp)`
- `doOptions(HttpServletRequest req, HttpServletResponse resp)`
- `doTrace(HttpServletRequest req, HttpServletResponse resp)`
- `doHead(HttpServletRequest req, HttpServletResponse resp)`
- `getLastModified(HttpServletRequest req)`
- `service(HttpServletRequest req, HttpServletResponse resp)`
- `service(ServletRequest req, ServletResponse res)`

Das untergeordnete `GenericServlet` hat folgende Methoden (welche dann ebenfalls in HttpServlet verfübar sind):

- getInitParameterNames(), getInitParameter(String name)
- getServletConfig(), getServletContext(),
- getServletInfo(), getServletName()
- log(String msg), log(String message, Throwable t)
- init(ServletConfig config)
- service(ServletRequest req, ServletResponse res)
- destroy()

Ein `HttpServletRequest` hat folgende Methoden (und mehr):

- String getParameter(String name) (ServletRequest)
- Cookie[] getCookies()
- Locale getLocale() (ServletRequest)
- String getQueryString()
- String getMethod()

`HttpServletResponse`:

- PrintWriter getWriter() (ServletResponse)
- void addCookie(Cookie cookie)
- String encodeURL(String url)
- void sendRedirect(String location)
- void sendError(int sc)
- void sendError(int sc, String msg)

Zuordnung zur URL:

Entweder per Annotation:

```java
@WebServlet("/MyServlet")
```

oder per Deployment Descriptor "web.xml"
