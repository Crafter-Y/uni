## Model 1 oder 2 Architektur

Aktuell befindet sich die Kernlogik in der View (JSP Datei), welche wenn nötig Beans importiert. Diese Konzept kann man als Model-1 Muster bezeichnen.

Der klassischere Ansatz wäre eine Trennung zwischen Model-View-Controller (MVC), hier bezeichnet als Model-2 Muster.

Der Client ruft dann nun eine Methode in einem Controller auf, welche die Daten in Beans aufbereitet oder aus einem DBMS ausliest und an das View übergibt, welches an den Client zurückgegeben wird.

Hier kommt wieder das Servlet zum Einsatz:

1. Daten im Request setzen

```java
request.setAttribute("address", myAddress);
```

2. An JSP weitergeben:

```java
response.sendRedirect("AddressDetail.jsp?id="+myAddress.getId());
// oder
request.getRequestDispatcher("AddressForm.jsp").forward(request, response);
```

3. Im JSP die Werte verwenden:

```java
request.getAttribute("address")
```

Hier bietet es sich an, Data Access Objects (DAOs) zu verwenden. Das sind Klassen, welche keine Logik, sondern nur Daten enthalten. 

Beispielsweise:

```java
public class Address {
    private long id;
    private String lastname;
    private String firstname;
    ...

    public Address() {
        this.id = 0;
        this.firstname = "";
        this.lastname = "";
        ...
    }
    public long getId() {
        return id;
    }
    public void setId(long id) {
        this.id = id;
    }
    public String getLastname() {
        return lastname;
    }
    ...
```
