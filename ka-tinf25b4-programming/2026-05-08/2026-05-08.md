## Abstrakte Klassen

Aus existierenden Klassen können spezialisierte Klassen abgeleitet werden. 

Eine abstrakte Klasse ist eine bewusst unvollständige Oberklasse, in der von einzelnen Methodenimplementierungen abstrahiert wird.

Java-Schlüsselwort `abstract`:

```java
public abstract class Graphic {
    abstract void draw();
}
```

Fehlende Methodenrümpfe werden erst in abgeleiteten Unterklassen implementiert. Java-Schlüsselwort `extends`.

## Interfaces

Interfaces definieren gleiche Schnittstellen in unterschiedlichen Klassen.

```java
public interface Graphic {
    public void draw();
}
```

Interfaces können dann von Klassen implementiert werden: Java-Schlüsselwort `implements`.

Seit Java 8 können Methoden in Intefaces auch eine Standardimplementation haben:

```java
public interface InterfaceA {
    default void methodA() {
        System.out.println("Invoked <InterfaceA>.methodA()");
    }
}
```

## Klassenbeziehungen

**Subobjekt**: ein Attribut einer Klasse

**Komposition**: Zusammensetzung eines Objektes aus mehreren Subobjekten

**Aggregation**: Loser Verbund von Subobjekten

**Delegation**: Prinzip der Implementierung einer Methode durch Weiterreichen eines Methodenaufrufs an ein Subobjekt

## Innere Klassen

Neben den "Top-Level-Klassen" gibt es in Java vier Typen von inneren Klassen:

**Statische innere Klassen**

Wenn die innere Klasse `static` ist, kann sie ohne die äußere Klasse instanziiert werden. Eine Referenz auf Instanzvariablen der äußeren Klasse ist nicht möglich.

**Mitglieds- oder Elementklassen**

Eine Mitgliedklasse (member class) kann hingegen auf die äußere Instanz zugreifen. 

Bei folgender Klasse:

```java
class House {
    class Room {
    }
}
```

Würde `Room` folgendermaßen instanziiert werden:

```java
Room room = new House().new Room();
```

**Lokale Klassen**

Innerhalb von Methoden kann man ebenfalls sogenannte "Lokale Klassen" erstellen, um sie an dieser Stelle zu verwenden. Diese stehen dann aber nur in Verbindung mit zum umliegenden Wertebereich (Scope). Methoden und `final` Variablen lassen sich also verwenden:

```java
public class Outer {
    public static void main(String args[]) {
        int i = 2;
        final int j = 3;
        class Inner {
            Inner() {
                System.out.println(j); // seen as final by the compiler
                System.out.println(i);
            }
        }
        new Inner();
    }
}
```

**Anonyme innere Klassen**

Anonyme Klassen gehen noch einen Schritt weiter als lokale Klassen. Sie haben keinen Namen und erzeugen immer automatisch ein Objekt.

```java
new ClassThatGetsExtended() {
    // implementation details, e.g. @Override to overwrite existing methods
};
```
