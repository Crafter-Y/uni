## Java 8 Features

**Effectively final**

```java
public void buildMyUi() {
    JButton b = new JButton(); // kein final Keyword!
    b.addActionListener(new ActionListener() {
        public void actionPerformed(ActionEvent e) {
            // in anonymer innerer Klasse verwendet
            System.out.println( b.getText() );
        }
    });
}
```

**Functional Interfaces**

Seit Java 8 gibt es folgende weitere Interfaces:

- Consumer `(void accept(T))`
- BiConsumer `(void accept(T,U))`
- Function `(R apply(T))`
- BiFunction `(R apply(T,U))`
- Supplier `(T get())`
- Predicate `(boolean test(T))`

**Lambda Expressions**

Man kann Instanzen einer Klasse auch in verkürzter Schreibweise instanziieren:

Anstelle von dem Ansatz mit den klassischen anonymen Methoden/Klassen:

```java
JButton b = new JButton();
b.addActionListener(new ActionListener() {
    public void actionPerformed( ActionEvent e ) {
        System.out.println( b.getText() );
    }
});
```

wird zu:

```java
JButton b = new JButton();
b.addActionListener(e -> System.out.println(b.getText()));
```

Ebenfalls sind komplette auslagerungen möglich durch eine referenz zur aufzurufenden Methode:

```java
// Old
Collections.sort(names, (a, b) -> this.compareByLength(a, b));

// New
Collections.sort(names, this::compareByLength);
```

**Streaming-API**

Ermöglicht sequenzeille und parallele Ausführung von Operationen auf Collections.

Steam starten:

```java
List<String> names = new ArrayList<>();

names.stream() // sequencial order
names.parallelStream() // parallel order
```

Nun gibe es verschiedene Methoden um die Daten zu verändern:

- `forEach`
- `limit`
- `filter`
- `map`
- `skip`
- `distinct`
- `peek`
- `flapMap`
- `sorted`
- `reduce`

Und am Ende kann man die Daten wieder zurück bekommen:

- `collect`
  - `Collectors.toList`
  - `Collectors.joining`
- `toList`
- `count`
- `findFirst`
- `findAny`
- `min`, `max`, `average`, `sum`

**NIO 2**

NIO 2 bietet die APIs `Files`, `Path` und `Paths`, welche wir zuvor schon verwendet haben.

**Records**

Record bietet eine Kurzschreibweise für Klassen, welche für Felder bereits einen AllArgsConstructor, Getter, Setter, hashCode und equals bereit stellt.

Folgende Klasse:

```java
public class Person {
    private String name;
    private String givenName;

    public Person(String name, String givenName) {
        this.name = name;
        this.givenName = givenName;
    }

    public int hashCode() { /* ... */ }
    public boolean equals(Object obj) { /* ... */ }
    
    public String getName() { return this.name; }
    public String getGivenName() { return this.givenName; }
}
```

hat nun diese Kurzschreibweise:

```java
public record Person (String name, String givenName) { }
```
