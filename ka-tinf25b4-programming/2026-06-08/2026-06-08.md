## Swing Komponenten anpassen

Um das Erscheinungsbild eines Swing Components zu bearbeiten, kann man folgende Methoden von `JComponent` implementieren:

- `paintComponent(Graphics)`
- `paintBorder(Graphics)`
- `paintChildren(Graphics)`

z.B. 

```java
public class CustomComponent extends JComponent {
    @Override
    protected void paintComponent(Graphics g) {
        g.drawString("Hello World", 25, 50);
    }

    // ...
}
```

Es ist auch möglich von einem JFrame direkt die `paint(Graphics)` Methode zu überschreiben.

Beim manuellen Zeichnen verwendet man immer ein Koordinatensystem, ausgehend vom oberen linken Rand.

In beiden Fällen bekommt man Zugriff auf die `Graphics` API. Diese erlaubt Zugriff auf folgende Methoden (Auswahl):

- `setFont(Font)`
- `setColor(Color)`
- `drawString(String, int, int)`
- `drawLine(int, int, int, int)`
- `drawRect(int, int, int, int)`
- `fillRect(...)`
- `drawOval(int, int, int, int)`
- `drawArc(int, int, int, int, int, int)`
- `fillArc(int, int, int, int, int, int)`
- `clearRect(int, int, int, int)`
- `copyArea(int, int, int, int, int, int)`
- `draw3DRect(int, int, int, int, boolean)`
- `drawPolygon(int[], int[], int)`
- `drawImage(Image, int, int, ImageObserver)`
- `setClip(int, int, int, int)`

## Maven

Maven ist ein **stark veraltetes** Build-Management-Tool der Apache Software Foundation und basiert auf Java. Mit ihm kann man insbesondere Java-Programme standardisiert erstellen und verwalten.

Für ein Projekt definiert man eine `pom.xml` Datei im Projektverzeichnis. Die POM (Project Object Model) Datei enthält i.d.R. alle Informationen zu dem Projekt.

Definiert werden:

- groupId: Oft eine Domain in Umgedrehter Form, z.B. `de.craftery`
- artifactId: Projektname, z.B. `uni`
- Version
- Repositories und Dependencies
- Plugins
- VCS, z.B. git

Maven hat mehrere Lifecycle Steps, welche teils miteinander verbunden sind. Man kann Abhängigkeiten und neue Steps auch manuell definieren:

| Phase    | Beschreibung                                                       |
| -------- | ------------------------------------------------------------------ |
| validate | Überprüfung der Projektstruktur auf Gültigkeit und Vollständigkeit |
| compile  | Java Compilation                                                   |
| test     | Ausführung der Unit-Tests                                          |
| package  | Zusammenpacken zu einem Produkt, z.B. JAR Datei                    |
| verify   | Überprüfen von Struktur oder Qualität                              |
| install  | Installation im Local Repository                                   |
| deploy   | Installation im Remote Repository                                  |

Eine `pom.xml` kann etwa so aussehen:

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0">
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.mycompany.app</groupId>
  <artifactId>my-module</artifactId>
  <version>1</version>

  <properties>
    <mavenVersion>3.0</mavenVersion>
  </properties>

  <dependencies>
    <dependency>
      <groupId>org.apache.maven</groupId>
      <artifactId>maven-artifact</artifactId>
      <version>${mavenVersion}</version>
    </dependency>
    <dependency>
      <groupId>org.apache.maven</groupId>
      <artifactId>maven-core</artifactId>
      <version>${mavenVersion}</version>
    </dependency>
  </dependencies>
</project>
```
