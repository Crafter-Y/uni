## Datenstrukturen

Datenstrukturen dienen dazu, anwendungsspeziische Informationen im Speicher abzubilden. Grundlegende Datenstrukturen gibt es bereits seit JDK 1.0. Die meisten Strukturen befinden sich in `java.util`.

Seit JDK 1.2 wird das Collection-Framework mitgeliefert, seit JDK 1.5 gibt es generics und seit java 1.8 Lambda expressions.

Die **Collection API** definiert folgende Schnittstellen, welche implementiert werden können: `List`, `Set`, `SortedSet`, `Map`, `Stack`

**List** definiert eine Ansammlung von Objekten und ähnelt dem array. Meistens werden die Implementationen `ArrayList` und `LinkedList` verwendet.

Um den Datentyp einer **generischen Klasse** festzulegen, wird dieser dem Klassennamen in Spitzen Klammern hinzugefügt:

```java
List<Integer> list = new ArrayList<>();
```

Alternativ kann man Listen auch so definieren:

```java
List<String> names = List.of("Bobby", "Peter", "Konrad");
```

Allerdings erhält man dann eine "immutable" Liste, also eine Liste, die nicht modifizierbar ist.
