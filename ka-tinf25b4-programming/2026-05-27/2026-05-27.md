## Generics

Generics sind ein Sprachmittel von Programmeirsprachen. In Java heißt das, dass man Klassen und Methoden mit Typparametern parametrisieren kann. 

Eine generische Klasse erstellt man folgendermaßen:

```java
public class MyClass<T> { // hier wird irgendein T angenommen
    private T value; // Kann innerhalb der Klasse verwendet werden

    // ...

    public void add(T value) {
        // ...
    }
}
```

**Wildcards**

Man kann auch einen unbekannten typ für einen generischen Typ setzen:

```java
List<Integer> intList = new ArrayList<>();
List<?> list = intList;
```

In die Liste `list` kann dann jedoch nur `null` eingefügt werden und per `List#get` kann nur `Object` ausgegelsen werden.

**Kovarianz und Kontravarianz**

Den diesen Wildcard Typen kann man aber auch einschränken:


```java
List<? extends Number> list = ...

Number n = list.get(0);
```

Dennoch kann weiterhin nur `null` hinzugefügt werden.

Andersherum kann man auch nach Oberklassen typisieren:

```java
List<? super Integer> list = ...;
```

Damit kann nur `Object` ausgelesen werden, aber nun `Integer` hinzugefügt werden.

## Enums

Enums:

- müssn explizit aufgezählt werden
- können als Konstante betrachtet werden
- Sind in ihrer Reihenfolge festgelegt

Der einfachste Enum Typ sieht folgendermaßen aus:

```java
enum MyEnum {
    FIELD1,
    FIELD2,
    FIELD3;
}
```

Enums haben die Standardmethoden `<T>[] values()`, `<T> valueOf(String value)` und `int ordinal()`

Enums kann man aber auch komplexer mit Daten gestalten:

```java
public enum MyAdvencedEnum {
    FIELD1(1, "Hello"),
    FIELD2(69, "Sailor"),
    FIELDXYZ(420, "Nice");

    private int easyNumber;
    private String description;

    MyAdvencedEnum(int easyNumber, String description) {
        this.easyNumber = easyNumber;
        this.description = description;
    }

    // ... weitere Methoden
}
```
