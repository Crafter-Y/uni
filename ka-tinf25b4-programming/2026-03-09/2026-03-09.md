## I/O

Es gibt in Java mehrere Wege, um mit Dateien umzugehen. `java.io` ist älter, basic und für manche Usecases veraltet. Es gibt mittlerweile aber `java.nio` (und mittlerweile NIO 2), was Dinge besser macht.

Verzeichnisse haben einen Namen und einen Elternpfad, beides zusammen ist der Verzeichnispfad.

Dateien werden über das Verzeichnis in dem sie gespeichert sind un ihrem Namen angesprochen, dem Dateipfad

Für Pfade gibt es 2 Varianten:

- Absolut
  - Beginnt bei Windows mit `$LAUFWERK:\`
  - unter UNIX mit einem `/`
- Relativ: beginnt mit einem Pfad relativ zur aktuellen Umgebung.

**File**

Eine File repräsentiert einen Datei oder Verzeichnis Pfadnamen, auch wenn sie noch nicht existieren.

z.B.

```java
File file = new File("tmp.txt");
```

**Path**

Ist ebenfalls eine Adresse zu einer Datei, jetzt aber mit NIO.

z.B:

```java
Path path = Paths.get("tmp.txt");
```

**Streams**

Für Ein- und Ausgabestreams hat java vier Basisklassen: `InputStream`, `OutputStream`, `Reader` und `Writer`

Input Stream:

- `read()` liest das nächste Byte vom Stream
- `read(byte[] b, int off, int len)`: liest mehrere Bytes
- `skip(long n)` überspringt n Bytes
- `close()` schließt den Stream.

Output Stream:

- `write(int b)` schreibt ein Byte
- `write(byte[] b, int off, int len)`: schreibt mehrere Bytes
- `flush()` 
- `close()`

Diese Klassen gibt es dann auch für Dateien:

|                 | Binäre Dateien   | Textdateien |
| --------------- | ---------------- | ----------- |
| Datei lesen     | FileInputStream  | FileReader  |
| Datei schreiben | FileOutputStream | FileWriter  |

**Try Resource**

Ressourcen müssen geschlossen werden, wenn sie nicht mehr gebraucht werden. Das kann man auch mit einem Try-Resource Block machen:

```java
try (InputStream is = new FileInputStream(source)) {
  // Streams are available here
  // Resources get closed automatically afterwards
} catch(IOException ex) {
  // Error handling
}
```

**Höherwertige I/O Klassen**

- `PrintStream`
- `PrintWriter`
- `BufferedInputStream`, `BufferedOutputStream`
- `BufferedReader`, `BufferedWriter`
