Primimplikanten können im KV-Diagram als Blöcke von Feldern identifiziert werden, die

1. alle "1"-Belegung aufweisen
2. rechteckige Form haben
3. eine 2er Potenz an Feldern umfassen
4. maximal groß sind

Kernprimimplikanten erkennt man im KV-Diagram daran, dass mindestens ein Feld von keinem anderen Primimplikanten überdeckt wird:

![f4-example.svg](./f4-example.svg)

Primimplikanten:

- 2+6: $\neg a b \neg d$ (KPI wegen 6)
- 2+10: $\neg a b \neg c$ (Kein KPI)
- 10+11: $b \neg c d$ (KPI wegen 11)
- 12: $\neg a \neg b c d$ (KPI wegen 12)

DMF: $\neg a b \neg d \lor b \neg c d \lor \neg a \neg b c d$

Hinweis: Die KPI reichen für eine DMF nicht aus, wir brauche noch einen einfachen PI. Hier gibt es sogar (mindestens) zwei zue Auswahl.

Hinweis: Trotz einachem PI in einer DMF kann die DMF trotzdem eindeutig sein, wenn:

- die zwei die fehlende "1" abdeckenden PI unterschiedliche Größe haben
- ein PI zwei fehlende Felder und zwei andere PI jeweils nur ein fehlendes Feld abdecken

**KNF & KMF vs. DNF & DMF**

| --- | --- | ---------- | -------------------- | --------------------------- |
| 0   | 000 | 1          | $\neg c\neg b\neg a$ |                             |
| 1   | 001 | 1          | $\neg c \neg b a$    |                             |
| 2   | 010 | 0          |                      | $c \lor \neg b \lor a$      |
| 3   | 011 | 1          | $\neg c b a$         |                             |
| 4   | 100 | 0          |                      | $\neg c \lor b \lor a$      |
| 5   | 101 | 0          |                      | $\neg c \lor b \lor \neg a$ |
| 6   | 110 | 0          |                      | $\neg c \lor \neg b \lor a$ |
| 7   | 111 | 1          | $cba$                |                             |

Primimplikate (Nicht Primimplikanten!):

- 4+5: $\neg c \lor b$ (Kernprimimplikat wg. 5)
- 2+6: $\neg b \lor a$ (Kernprimimplikat wg. 2)
- 4+6: $\neg c \lor a$ (kein Kernprimimplikat)

KNF: $(c \lor \neg b \lor a) \land (\neg c \lor b \lor a) \land (\neg c \lor b \lor \neg a) \land (\neg c \lor \neg b \lor a)$

KMF: $(\neg c \lor b) \land (\neg b \lor a)$

Vorteil KNF vs DNF: bei weniger "0" als "1" in der Ausgangsbelegung, also weniger Maxterme als Minterme folgt: Weniger Hardwareaufwand für KNF als für DNF.

## Schaltwerke

Schaltwerke bestehen wie Schaltnetze nur aus Gattern und Verbindungsleitungen. Allerdings sind bei einem Schaltwerk (im Gegensatz zu Schaltnetzen) auch Rückkopplungen von Aus- zu Eingängen erlaubt. Diese führen zu einem Speicherverhalten (welches ein Schaltnetz als Realisierung einer Funktion nicht haben darf).

![rsff.svg](./rsff.svg)

- $y = \neg(b \lor x')$

**Kochrezept Schaltwerksanalyse**

1. Auftrennen der rückgekoppelten Eingänge mit Neubenennung
2. Aufstellen von Ausdrücken der Schaltalgebra für Ausgümnge und der Wertetabelle, wobei die vormals rückgekoppelten Eingänge ganz links notiert werden
3. Markieren von stabilen und instabilen Zeiten der WT. Bei stabilen Zeiten stimmen die Ausgangsbelegungen mit dem entsprechenden (rückgekoppelten) Eingangsbelegungen überein
4. Notieren der Folgezeilen bei instabilen Zeilen, bis entweder eine stabile Zeile oder ein "Kreislauf" erreicht ist
5. Bennenen der Zustände anhand der Belegungen der rückgekoppelten Eingänge
6. Aufstellen des Zustandsübergangsdiagramms, wobei Zustandsübergänge abhängig von den Belegungen der unabhängigen (nicht rückgekoppelten) Eingänge passieren
7. Interpretaion des Zustandsübergangsdiagramms, um ein "sinnvolles Verhalten" herauszulesen

**RS-FF** (Reset/Set Flip-Flop)

- 1bit Speicherbaustein
- 4 Zustände, davon (nur) 2 Arbeitszustände:
  - Rückgesetzt ("0") 01
  - Gesetzt ("1") 10
- Unerwünschter Zustand 00
- Ein nie stabil erreichter Zustand 11
- 2 Eingänge: Reset (R) und Set (S)
  - R=0 und S=1 => gesetzter Zustand
  - R=1 und S=0 => rückgesetzter Zustand
  - R=S=0 => Arbeitszustand bleibt erhalten (bzw. "Zustandsflimmern" bei 00, deshalb unerwünschter Zustand 00)
  - R=S=1 => verbotene Eingangsbelegung, da der unerwünschte Zustand erreich würde.
- 2 Ausgänge:
  - Q: Zustand des Schaltwerks
  - Q*: invertierter Zustand des Schaltwerks (gilt in den beiden Arbeitszuständen)

Praktische Anwendung:

- Schalter mit zwei unterschiedlichen Schaltstellen für "An" und "Aus", also z.B. "Not-Aus" an R bei Maschinen
- Treppenbeleuchtung (S durch benutzer, R durch Zeitschalter)

![rsff-symbol.svg](./rsff-symbol.svg)

**D-FF** (Data Flip-Flop)

- Ebenfalls ein 1bit Speicher, aber mit nur einem Eingang sowohl zum setzen als auch zum Rücksetzen
- Eingang: D ("Data")
  - D=1: FF gesetzt
  - D=0: FF rückgesetzt

![dff-symbol.svg](./dff-symbol.svg)

Das bringt so aber noch nichts, da es das selbe wäre, wie wenn man einfach die Datenleitung mit dem Output (Q) verbindet.

**Taktsteuerung**

Takt ist ein weiterer Eingang der Schaltung, der isch aber nicht auf die eigentliche Funktionalität der Schaltung auswirkt, sondern nur bestimmt, wann die anderen Eingänge dies tun können.

Aktiver Takt: Eingansgbelegung hat unmittelbar Auswirkung auf Ausgangsbelegung und/oder Zustand

Inaktiver Takt: Eingangsbelegung hat keine Auswirkung, die Schaltung behält die Ausgangsbelegung (und auch den Zustand) beim letzten aktiven Takt bei.

=> Eine Schaltung mit Taktsteuerung ist immer ein Schaltwerk. Ein Schaltnetz wird mit Taktsteuerung immer zum Schaltwerk.
