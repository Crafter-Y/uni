## Bitfehler erkennen

Die Bitfehlerquote sagt nicht über die Verteilung von Bitfehlern aus.

- diese können zwar statistisch gleichverteilt auftreten,
- doch sind ebenso so genannte Bursts möglich, bei dem Fehler gehäuft direkt in zeitlicher Nähe auftreten

Gerade bei Funkübertragungen oder auf physikalischen Speichern treten Bursts in der Form von Störsignalen ... häufiger auf.

Ein Interleaving bezeichnet die gleichmßige Streuung der Kodierungsbits auf einen größeren Bereich. Dazu werden die Daten zeilenorientiert in einen definierten Block geschrieben und spaltenorientiert wieder ausgelesen.

**Parity-Verfahren**

Eine gerade **Parität** ergänzt einen Datentyp um einen Bit, so dass die Summe aller Bits in $\mathbb{Z}_2$ gerade 0 ergibt, die ungerade Parität ergänzt den Datentyp um einen Bit, so dass die Summer aller Bits in $\mathbb{Z}_2$ genau 1 ergibt.

Zur Berechnung der **Blockparität** werden $n \cdot m$ Bits in eine Matrix $\mathbb{Z}^{n \times m}$ zeilenorientiert geschrieben und für jede Zeile und jede Spalte insgesamt $n+m$ Paritätsbits bestimmt.

Zur Berechnung der **Dreiecksparität** werden $\frac{n \cdot (n+1)}{2}$ Bits in eine linke untere Dreiecksmatrix geschrieben und $n+1$ Paritätsbits aus an der nächsten Diagonale angrenzenden Zeile und Spalten berechnet.

**Hamming Distanz**

Um den Unterschied zwischen korrekten Daten und gestörten Daten bezeichnen zu können, wird ein Begriff für den Grad der Änderung benötigt.

Sei $n \in \mathbb{N}$ und seien $x,y \in \mathbb{Z}_2^n$ zwei Binärvektoren der Länge $n$. Dann bezeichnet die Hammingdistanz $H(x,y)$ die Anzahl der unterschiedlichen Bits:

$$
H(x,y) = \sum_{k=1}^n \begin{cases}
0, \text{ wenn } x_k = y_k \\
1, \text{ wenn } x_k \neq y_k
\end{cases}
$$

Über die geringste Hammingdistanz zu Codewörtern können Fehler korrigiert werden. Haben die Codewörter eines Codes untereinander eine minimale Hammingdistanz von 3, so können einfache Bitveränderungen, also empfangene Daten mit einer Hammingdistanz von 1 zu einem Codewort ...

> Wir wählen $p(x)$ (mit $dig(p) = k$) mit dem Codewörter berechnet werden können.
> 
> Der generierte Code soll aber linear und Zyklisch sein (für einfache Beschreibung und Fehleranalyse) 
> 
> $\rightarrow$ Daher muss $p(x)$ folgende Eigenschaften erfüllen:
> 
> - $p(x)|x^N+1$ (können wir durch Polynomdivision prüfen)
>   
> Beispiel: $P(x) = x^3 +  x + 1$ ist ein Generatorpolynom in $\mathbb{Z}_2[x]\backslash_{x^7+1}$

> $$
>  (x^7               +1) : (x^3 + x + 1) = x^4+x^2+x+1
> -(x^7+x^5+x^4) x^3 
>       x^5+x^4         +1
>     -(x^5+   x^3+x^2)
>          x^4+x^3+x^2  +1
>        -(x^4+    x^2+x)
>              x^3    +x+1
>            -(x^3    +x+1)
>                        0
> $$

3. Wir wollen systematische Codes $s(x)$ erzeigen, s.h. die Datenbits solleni sichtbar bleiben.

$\rightarrow$ Wir hängen $k = deg(p)$ freie Stelleen an die Daten $q(x)$ an und berechnen Prüfbits $r(x)$, sodass das gesamte Wort durch $p(x)$ teilbar ist.

$$
s(x) = q(x) \cdot x^k + r(x) \rightarrow r(x) \text{ ist Rest bei Division von } q(x) \cdot x^k \text{ mit } p(x)
$$

Beispiel: Daten $(0,1,0,1): q(x) = 0 \cdot x^3 + x^2 + 0 \cdot x + 1$

$q(x) \cdot x^k = (x^2+1) \cdot x^3 = x^5 + x^3$

$$
 (x^5 + x^3) : (x^3 + x + 1) = x^2
-(x^5 + x^3 + x^2)
              x^2 \rightarrow r(x) = x^2, \text{ Prüfbits } (1,0,0)
$$

$\Rightarrow$ Codewort: $s(x) = x^5 + x^3 + x^2$

$\Rightarrow \underbrace{0 1 0 1}_{Daten} \underbrace{1 0 0}_{Prüfbits}$
