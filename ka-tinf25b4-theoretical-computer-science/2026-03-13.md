## Haskell Funktionen II

Die Funktion zur Multiplikation von Werten mit 3 lautet `(*) 3` und die Additions mit eins lautent sntsprechend `(+) 1`. Entsprechend kann die Funktion, die mit drei multipliziert und anschließend eins addiert so geschrieben werden und einem Platzhalter als neue Funktion zugewiesen werden:

`mal3plus1 = (((+) 1) . ((*) 3))`

Einfach Funktionsdefinitionen bestehen aus Typbeschreibung von Definition gefolgt von :: und dem Typ, gefolgt von dem Identifier, dann Parameter-Variablen, Gleichheitszeichen und der Implementation:

```haskell
ident :: Typ -> ... -> Tyü
ident x_1 ... x_n = ...
```

Wenn die Implementation die Typ-Signatur hinreichend genau beschreiben kann, kann die Typbezeichnung ausbleiben. In der Typbeschreibung sind auch Variablen erlaubt. Eine Implementation kann aus mehreren Zeilen bestehen, wo bestimmte Instanziierungen von Variablen vorgegeben sind, von denen die erste passende gewählt wird.

**Guards**

Mit Guards können Funktionen mit Fallunterscheidungen definierrt werden:

```haskell
ident :: Typ -> ... -> Typ
ident x_1 ... x_n
  | cond_1 x_1 ... x_n = compute_1 cond_1 x_1 ... x_n
  ...
  otherwise = compute_0 cond_1 x_1 ... x_n
```

**Tupel**

Ein Tupel ist ein Verbund fester Länge von Ausdrücken:

`(ident_1, ..., ident_n)`

mit Typ `(Typ_1, ..., Typ_n)` wenn `ident_k` den Typ `Typ_k` besitzt.

Für Tupel der Länge 2, also Paare, gibt es die Funktion `fst` und `scd` um das erste oder zweite Element zu bekommen.

**Liste**

Eine Liste ist ein Verbund von Ausdrücken gleichen Typs, auf dessen Elemente nach Index und vielen weiteren Methoden zugegriffen werden kann:

`[ident_1, ..., ident_n]`

Die Liste hat den Typ `[Typ]`, wenn alle `ident_k` den Typ `Typ_k` haben.

**Listenerzeugung**

Mit dem `..` Operator kann man Listen erzeugen:

- Unendliche Liste mit Startwert: `[a..]`
- Endliche Liste mit Endwert: `[a..e]`
- Endliche Liste mit Schrittweite und Endwert: `[a,b..e]`

`repeat` liefert eine Liste mit unendlich vielen Einträgen des ersten Arguments.
