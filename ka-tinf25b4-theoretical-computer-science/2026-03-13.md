## Haskell Funktionen II

Die Funktion zur Multiplikation von Werten mit 3 lautet `(*) 3` und die Addition mit Eins lautet entsprechend `(+) 1`. Entsprechend kann die Funktion, die mit Drei multipliziert und anschließend Eins addiert so geschrieben werden und einem Platzhalter als neue Funktion zugewiesen werden:

`mal3plus1 = (((+) 1) . ((*) 3))`

Einfache Funktionsdefinitionen bestehen aus Typbeschreibung von Identifier gefolgt von :: und dem Typ, gefolgt von dem Identifier, dann Parameter-Variablen, Gleichheitszeichen und der Implementation:

```haskell
ident :: Typ -> ... -> Typ
ident x_1 ... x_n = ...
```

Wenn die Implementation die Typ-Signatur hinreichend genau beschreiben kann, kann die Typbeschreibung ausbleiben. In der Typbeschreibung sind auch Variablen erlaubt. Eine Implementation kann aus mehreren Zeilen bestehen, wo bestimmte Instanziierungen von Variablen vorgegeben sind, von denen die erste passende gewählt wird.

**Guards**

Mit Guards können Funktionen mit Fallunterscheidungen definierrt werden:

```haskell
ident :: Typ -> ... -> Typ
ident x_1 ... x_n
  | cond_1 x_1 ... x_n = compute_1 cond_1 x_1 ... x_n
  ...
  otherwise = compute_0 cond_1 x_1 ... x_n
```

Beispiel:

```haskell
max3 :: Double -> Double -> Double -> Double
max3 a b c
  | a>=a && a>=c = a
  | b>=a && b>=c = b
  | otherwise = c
```

```haskell
mid3 :: Double -> Double -> Double -> Double
mid3 a b c
  | (a<b) && (a>c) = a
  | (a>b) && (a<c) = a
  | ((b>a) && (b<c)) || ((b>c) && (b<a)) = b
  | otherwise = c
```

```haskell
coll :: Int -> Int
coll x
  | even x = div x 2
  | otherwise = 3 * x + 1
```

**Tupel**

Ein Tupel ist ein Verbund fester Länge von Ausdrücken:

`(ident_1, ..., ident_n)`

mit Typ `(Typ_1, ..., Typ_n)` wenn `ident_k` den Typ `Typ_k` besitzt.

Für Tupel der Länge 2, also Paare, gibt es die Funktion `fst` und `snd` um das erste oder zweite Element zu bekommen.

`fst (1,2)` => 1

`snd (1,2)` => 2

**Liste**

Eine Liste ist ein Verbund von Ausdrücken gleichen Typs, auf dessen Elemente nach Index und vielen weiteren Methoden zugegriffen werden kann:

`[ident_1, ..., ident_n]`

Die Liste hat den Typ `[Typ]`, wenn alle `ident_k` den Typ `Typ_k` haben.

Der Doppelpunkt-Operator : fügt ein Element an den Kopf der Liste an:

`4:[2,5,1]` => `[4,2,5,1]`

Der Plusplus-Operator `++` fügt zwei Listen zusammen:

`[1,2,3] ++ [4,5,6]` => `[1,2,3,4,5,6]`

Die Funktionen `head`,  `tail`, `last` und `init` liefern das erste, alle außer das erste, das letzte und alle außer das letzte Element.

`length` liefert die Länge der Liste.

`length [1,2,3]` => 3

`null` ist genau dann wahr, wenn die Liste leer ist:

`null []` => `True`

`reverse` dreht eine Liste um.

**List Patterns**

- `ident [] = ...` Leere Liste
- `ident [x] = ...` Liste mit einem Element
- `ident (x:xs) = ...` Nimmt das erste Element x aus einer Liste und gibt die restliche Liste xs
- `ident (x1:x2:xs) = ...` Nimmt 2 Elemente aus der Liste

Beispiel:

```haskell
halfEven :: [Integer] -> [Integer]
halfEven [] = []
halfEven (x:xs)
  | (mod x 2)==0 = (div x 2):( halfEven xs )
  | otherwise = halfEven xs
```

```haskell
main :: IO ()
main = do
  print ( mittlere [1,5,9,5,1,4,8,0,3,1,45,8,0,5,1,5])

mittlere :: [Integer] -> [Integer]
mittlere [a,b] = []
mittlere (a:b:c:rest)
  | (a<b && b<c) || (c<b && b<a) = b:mittlere(c:rest)
  | otherwise = mittlere(b:c:rest)
```

**Listenerzeugung**

Mit dem `..` Operator kann man Listen erzeugen:

- Unendliche Liste mit Startwert: `[a..]`
- Endliche Liste mit Endwert: `[a..e]`
- Endliche Liste mit Schrittweite und Endwert: `[a,b..e]`

`repeat` liefert eine Liste mit unendlich vielen Einträgen des ersten Arguments.

Teile von Listen: `drop` überspringt die Anzahl Elemente im ersten Argument und gibt den Rest zurück, `take` übernimmt bis zu Anzahl Werte aus dem ersten Argument aus der Liste.

Aufteilen von Listen: `splitAt` teilt die Liste an Position des ersten Arguments auf, `break` teilt eine Liste im zweiten Argument, sobald die Funktion im ersten Argument wahr wird.

Elementesuche: `elem` und `notElem` prüfen, ob das erste Argument in der Liste auftritt.

Arithmetik: `sum` und `product` berechnen Summe oder Produkt aller Werte in der Liste.

Tupelerzeugung: `zip` erzeugt aus zwei gleichlangen Listen eine Liste mit Tupel der Elemente an gleicher Position, `unzip` zerteilt Tupel in zwei Listen, und gibt die beiden als Liste zurück.
